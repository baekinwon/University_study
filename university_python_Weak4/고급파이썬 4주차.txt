조건문

- 조건을 따져서 서로 다른 동작을 하도록 하는 것은 프로그램에서 매우 중요

- 프로그래밍에서 조건을 따지는 방법
  1) 질문 후 결정

- if문
  1) 조건이 참이면 명령문을 수행
  2) 조건이 거짓이면 명령문을 무시하고 다음 문장으로 이동

- if else 문
  1) 조건식이 맞으면 if 틀리면 else문 실행
  2) 상황이 2가지로 분명하게 나뉠 때 사용 용이

- block(코드 블록)
  1) 코드 덩어리(코드 집합)
  2) 어떤 조건 만족하는 경우 특정 코드 선택적 실행
  3) 들여쓰기 통해 지정
  4) 동일 코드 블록에서 들여쓰기의 칸 수 일치 필수
  5) 일반적으로 띄어쓰기 4번 주로 사용
  6) 띄어쓰기와 탭이 공존하면 에러발생 ★

- 중첩조건문
  1) if문을 한번 실행한 후 그 결과에서 if문을 다시 실행하는 것
  2) 조건문 내부에 또 다른 조건문이 존재 가능
  3) 좋지는 않지만 실무에서 어쩔수 없이 많이 사용하긴 함
  4) 최소한으로 사용할 것

- 논리 연산자
  1) 여러 개의 조건을 조합하여 참인지 거짓인지를 따질 때 사용
  2) 복잡한 조건의 표현에 사용 ( and, or, not )

- if elif else문
  1) 중첩조건문을 간략화 한 구조

관계 연산자

- 참과 거짓
  1) 명제 : 참 또는 거짓을 명확하게 판단 할 수 있는 문장

- 조건식
  1) 조건에 해당하는 부분을 수식으로 나타낸 것

- 관계연산자
  1) 두 수의 관계에 대해서 판단하는 연산자
  2) 참 또는 거짓의 결과를 갖는다.
